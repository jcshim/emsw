# 📘 **임베디드 소프트웨어 개인 종합과제 안내서**

## 1. 과제 목적

본 과제의 목적은 **Arduino–서버–스마트폰 앱**을 연동하여
**모니터링(센싱)** 과 **제어(액추에이션)** 기능을 모두 갖춘 **완성형 임베디드 시스템**을 직접 설계·구현하는 데 있습니다.

학생들은 센서와 액추에이터를 다루고, 서버를 통한 데이터 송수신, 스마트폰 앱과의 연동까지 포함한 **작은 IoT 프로젝트 전체 흐름**을 경험하게 됩니다.

---

## 2. 수행 요구사항

아래 항목을 반드시 포함해야 합니다.

### ✔ Arduino 기반 임베디드 장치

* 예: 온습도 센서, 조도 센서, 스위치 입력 등
* 제어: LED, 부저, 서보모터, 릴레이, 피에조 스피커 등

### ✔ 서버 프로그램 활용

* 예: Processing 서버
* 역할: Arduino ↔ 앱 간 데이터 중계 및 저장

### ✔ 스마트폰 앱 활용

* 예: App Inventor, Flutter, Android Studio 등
* 역할: 모니터링 화면(센서값 표시), 제어 버튼(LED/알람 등)

### ✔ 기능 요구

* **모니터링 기능 1개 이상**
* **제어 기능 1개 이상**
  (예: 온도 확인 + 팬 ON/OFF)

## 4. 진행 절차 (권장 일정)

### **1) 프로젝트 주제 선정**

* 본인의 흥미·생활편의·창의성 등 고려하여 자유롭게 선택

### **2) 요구사항 정의 → 과제 계획서 작성**

* 어떤 센서를 사용할지? 어떤 기능을 제공할지?
* 데이터 흐름(Arduino↔Server↔App) 구상

### **3) 모듈별 기능 구현**

* Arduino 기능 테스트
* 서버 통신 테스트
* 스마트폰 앱 UI/기능 테스트

### **4) 전체 시스템 통합**

* 센서값 → 서버 → 앱 표시
* 앱 → 서버 → Arduino 제어 신호 송신
* 오류 수정 및 디버깅

### **5) 시연 영상 제작 & 보고서 작성**

* 기능 실행 장면 포함
* GitHub 저장소 정리 후 제출

## 6. 프로젝트 예시 주제

가능하면 직접 새로운 주제 시나리오를 만들어 주세요. 혹시 선정할 만한 주제가 없으면 다음 중에 선택해도 됩니다.

* **스마트 도어락**
  (문 열림 감지 + 원격 잠금/해제)

  (조명/알람/난방 제어 기능 일부)
* **환경 모니터링 시스템**
  (온습도·미세먼지 모니터링 + 경보)
* **스마트 환풍기 제어**
  (주변 온도·가스 측정 → 앱 제어)
